    Checklist de Desenvolvimento Web

    Nessa fase, é importante garantir que a equipe de desenvolvimento saiba como implementar acessibilidade, verificá-la e usar as ferramentas disponíveis no mercado para testes e padronizações. Também é importante que a equipe saiba como as pessoas com deficiência usam os sítios web e aplicativos". Nesse caso, foram incluídas técnicas relacionadas ao desenvolvimento e também formas de verificar

    Imagens

    • Adicionar alt para imagens, botões-imagem, gráficos e imagens de mapas com pontos de acesso. [1] [A] [NBR 5.2.4]

    • Adicionar alt="" para imagens decorativas que não tem significado. [1] [A] [NBR 5.2.3]

    • Incluir a descrição no alt para imagens que contém texto. [1] [A] [NBR 5.2.5]

    Vídeos e Mídia

    • Adicionar legendas para áudio em vídeos existentes. [2] [A] [NBR 5.14.2]

    • Impedir autoplay para vídeos e audios. [12] [A] [NBR 5.14.7]

    • Adicionar pausa para todas as mídias. [13] [A] [NBR 5.14.7] [NBR 5.15.1]

    • Adicionar transcrição para audios. [1] [A] [NBR 5.14.3]

    • Evitar conteúdo com flashes (gatilhos para convulsões) ou manter abaixo dos limites. [3] [A] [NBR 5.15.4]

    Controles e Interação

    • Adicionar href para links. [4] [A] [NBR 5.7.1]

    • Adicionar underline nos links. [5] [A] [NBR 5.11.1]

    • Adicionar estados de foco em campos de entrada, botões, e elementos interativos. [6] [AA] [NBR 5.1.1]

    • Adicionar type="button" nos botões. [4] [A] [NBR 5.8.1]

    • Adicionar skip-link (link para pular) para o conteúdo principal. [7] [A] [NBR 5.7.12]

    • Identifique e comunique links que abrem em uma nova guia ou janela. [8] [NBR 5.7.6]

    • Funcionalidades da página web estão disponíveis por teclado. [13] [A] [NBR 5.1.13]

    • Foco visível - remova elementos focalizáveis ​​invisíveis. [6] [AA] [NBR 5.1.1]

    • Adicione o .hover, .focus { } para tornar o foco visível. [6] [AA] [NBR 5.1.1]

    • Permite/visa o uso de Atalhos de teclado como o TAB. [13] [A] [NBR 5.1.13]

    • Primeiro item interativo da página é um link para o conteúdo principal. [7] [A] [NBR 5.7.12]

    Formulários

    • Adicionar label para os campos de entradas associadas ao elemento correspondente. [9] [A] [NBR 5.9.1]

    • Adicionar fieldset e legend para seção no formulário. [4] [A]

    • Adicionar autocomplete para campos de entrada. [10] [AA]

    • Exibir errors (erros) de entrada acima do formulário, após envio. [11] [A] [NBR 5.9.9]

    • Adicionar aria-describedby para os campos de entrada. [11] [A]

    • Exibir mensagens de erro e sucesso não só visualmente. [5] [A] [NBR 5.9.9] [NBR 5.11.1]

    Semântica e Estrutura

    • Uso de elementos nativos HTML. [NBR 5.13.11]

    • Fluxo continuo e Lógico. [NBR 5.10.2]

    • Tem descrições que podem ser facilmente compreendidas. [NBR 5.12.12]

    • Tem a semântica correta. [NBR 5.13.12]

    • É objetivo nos rótulos.

    • Não pular níveis lógicos de cabeçalho. [NBR 5.3.5]

    Texto

    • Evitar o uso de textos dentro de imagens. [NBR 5.2.5]

    • Redimensiona os textos na página, aumentando o zoom em até 200%. [2] [AA] [NBR 5.12.7]

    • Alturas das fontes não é fixa.

    Títulos e Cabeçalhos

    • A hierarquia de conteúdo da página é definida por sua lógica não pelo tamanho do texto. [17] [AA]

    • Use elementos de título h1 h2 h3 para apresentar o conteúdo. [17] [AA]

    • Toda página contem um título h1 descrevendo a página. [17] [AA]

    Tabelas

    • Use o table para elementos em formato de tabela. [4] [A] [NBR 5.6.1] [NBR 5.6.2]

    • Insira cabeçalhos para explicar os dados, use th com scope correto. [15] [NBR 5.6.3]

    • Use o caption elemento para fornecer um título para a tabela. [17] [AA] [NBR 5.6.4]

    Modais

    • Deve ser fácil fechar.

    • Permiti o uso da tecla escape ESC.

    • A interação é uma tarefa simples.

    • Evita modais em tela cheia. [NBR 5.10.4]

    • Não abrir um modal a partir de outro modal.

    Dispositivo Móvel e Touch

    • O site pode ser rotacionado para qualquer orientação. [14] [AA] [NBR 5.10.3]

    • Impedir rolagem horizontal. [16] [AA] [NBR 5.10.4]

    • Garantir que botões e links possam ser ativados facilmente. [18] [AAA] [NBR 5.8.7]

    • Garantir espaço suficiente entre elementos interativos. [7] [A] [NBR 5.8.7]

    Ferramentas e Extras

    • Usar Breadcrumbs informando a localização atual nas páginas.

    • Colocar página ou área de esclarecimento de dúvidas e dicas de acessibilidade. [NBR 5.9.13]

    • Áreas clicáveis com no mínimo 44px (pixels) de altura e 44px de largura. [18] [AAA]

    • No caso de captcha garanta que seja simples de entender e tenha alternativas para pessoas com deficiência. [1] [A] [NBR A.1.1]

    • Incluir um campo de busca. [NBR 5.7.13]
